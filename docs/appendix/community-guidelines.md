# Community Guidelines

> Standards for behavior, communication, and collaboration within the Awesome Vibecoding Guide community.

## Code of Conduct

### Our Pledge

We as members, contributors, and leaders pledge to make participation in our community a harassment-free experience for everyone, regardless of:
- Age, body size
- Visible or invisible disability
- Ethnicity, sex characteristics, gender identity and expression
- Level of experience, education, socio-economic status
- Nationality, personal appearance, race, religion
- Sexual identity and orientation

We pledge to act and interact in ways that contribute to an open, welcoming, diverse, inclusive, and healthy community.

### Our Standards

**Positive Behaviors:**

| Behavior | Example |
|----------|---------|
| Welcoming language | "Welcome!" and "Great contribution!" |
| Respectful disagreement | "I see your point, and here's another view..." |
| Graceful acceptance | "Thanks for the feedback, I'll update" |
| Focusing on content | "This code could be improved by..." |
| Apologizing mistakes | "I made an error, let me fix that" |
| Seeking understanding | "Can you help me understand your approach?" |

**Negative Behaviors (Not Accepted):**

| Behavior | Example |
|----------|---------|
| Harassment | Unwelcome comments, personal attacks |
| Trolling | Deliberately provocative statements |
| Insults | Ad hominem attacks, put-downs |
| Discrimination | Slurs, hateful language |
| Unwelcome attention | Stalking, inappropriate contact |
| Doxing | Sharing private information |

### Enforcement Responsibilities

**Who Enforces:**

| Role | Responsibility |
|------|----------------|
| Maintainers | Immediate action on violations |
| Community Leads | Escalation handling |
| Project Owner | Final appeals |

**Enforcement Actions:**

| Violation | Action | Duration |
|-----------|--------|----------|
| First minor | Private warning | None |
| First major | Public warning | 7-day ban |
| Repeated minor | Temporary ban | 30 days |
| Severe | Permanent ban | Permanent |
| Legal issue | Immediate removal | Permanent |

**Reporting Process:**

1. Document the incident
2. Report via GitHub Issues (private)
3. Maintainers investigate within 48 hours
4. Action taken and communicated

## How to Ask for Help

### Before Asking

**Try These First:**

1. **Search existing issues** - Has this been asked?
2. **Check documentation** - Is it already covered?
3. **Search Discord/Forum** - Previous discussions
4. **Try it yourself** - Experiment first
5. **Read error messages** - They often contain hints

### Asking Effectively

**Good Questions Include:**

| Element | Example |
|---------|---------|
| Context | "I'm building a website for a local restaurant" |
| What you tried | "I used the AI prompt from section 2.3" |
| What happened | "The code generated 3 errors on line 45" |
| What you need | "I need help understanding the error" |

**Question Template:**

```markdown
## Question: [Brief Title]

### Context
[What you're trying to accomplish]

### What I've Tried
1. [First attempt]
2. [Second attempt]
3. [Third attempt]

### The Problem
[Error message or unexpected behavior]

### Environment
- Tool: [AI tool + version]
- Platform: [OS, etc.]
- Relevant code: [Brief snippet]

### What I Need
[Specific help you're looking for]
```

### Where to Ask

| Question Type | Best Channel |
|---------------|--------------|
| Bug report | GitHub Issues |
| Feature request | GitHub Discussions |
| Quick help | Discord #questions |
| In-depth discussion | GitHub Discussions |
| Security concern | Private email |

### Responding to Help

**As the asker:**

- Thank responders
- Share the solution if you find one
- Update the issue if resolved
- Pay it forward by helping others

**As the responder:**

- Be patient and kind
- Ask clarifying questions
- Provide clear answers
- Point to resources, don't just say "RTFM"

## Handling Disagreements

### Disagreement is Normal

Different perspectives improve the project. Handle disagreements constructively.

### Guidelines for Disagreement

**DO:**

- [ ] Focus on ideas, not people
- [ ] Present evidence for your view
- [ ] Listen to understand, not to respond
- [ ] Seek common ground
- [ ] Be willing to compromise
- [ ] De-escalate when heated

**DON'T:**

- [ ] Make it personal
- [ ] Assume bad intent
- [ ] Drag in unrelated issues
- [ ] Resort to insults
- [ ] Go silent or ghost
- [ ] Escalate prematurely

### Resolution Process

**Step 1: Direct Discussion**
- Discuss in the PR/issue thread
- Keep it focused on the topic
- Seek to understand the other view

**Step 2: Third-Party Input**
- Ask for maintainer opinion
- Reference similar past decisions
- Present alternatives

**Step 3: Formal Decision**
- Maintainer makes ruling
- Document the decision
- All parties accept outcome

### Escalation Path

| Level | Contact | For |
|-------|---------|-----|
| 1 | PR comments | Technical disagreements |
| 2 | GitHub Discussions | Content disagreements |
| 3 | @maintainers | Unresolved conflicts |
| 4 | project@email.com | Formal complaints |

## Mentor/Mentee Program

### Program Overview

Experienced contributors can mentor newcomers, accelerating learning and community integration.

### Becoming a Mentee

**Benefits:**

| Benefit | Description |
|---------|-------------|
| Guidance | One-on-one support |
| Faster learning | Skip common mistakes |
| Network | Connect with experienced devs |
| Opportunities | Early access to projects |

**Requirements:**
- New to the project (<5 contributions)
- Willing to learn
- Available for regular check-ins
- Respectful of mentor's time

### Becoming a Mentor

**Benefits:**

| Benefit | Description |
|---------|-------------|
| Give back | Help new contributors |
| Leadership | Build management skills |
| Recognition | Mentor badge, promotion credit |
| Network | Connect with mentees |

**Requirements:**
- Expert tier or higher
- 6+ months active
- Good communication skills
- Patient and supportive

### Program Structure

**Onboarding (Mentee):**
1. Apply via GitHub issue
2. Matched with mentor
3. Initial goal-setting call
4. Weekly check-ins for 8 weeks
5. Program completion

**Mentor Commitment:**
- 1 hour/week minimum
- Respond within 48 hours
- Provide constructive feedback
- Champion mentee's work

### Mentor/Mentee Matching

**Matching Criteria:**

| Factor | Weight |
|--------|--------|
| Primary interest match | High |
| Timezone overlap | Medium |
| Communication style | Medium |
| Experience gap | Low |
